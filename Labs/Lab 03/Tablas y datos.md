## Tabla de donde usar prio y tpicked


| Funcion   | tpicked | prio | Motivo                                                                         |
| --------- | ------- | ---- | ------------------------------------------------------------------------------ |
| procinit  | No      | No   | Declaramos tpicked y prio en *allocproc*                                       |
| allocpid  | No      | No   | IDEM que anterior                                                              |
| allocproc | Si      | Si   | Es donde los declaramos                                                        |
| freeproc  | Si      | Si   | Se liberan los procesos y la informacion que tenian                            |
| userinit  | No      | No   | Prepara el primer proceso, no nos interesa trabajar con este                   |
| fork      | No      | No   | No debemos copiar estos valores, perderian sentido                             |
| exit      | No      | No   | No tendria sentido usarlos aca ya que se sale del proceso dejandolo en zombi   |
| wait      | No      | No   | Porque depende del anterior                                                    |
| scheduler | Si      | No   | Actualizamos la cantidad de veces que un proceso es seleccionado               |
| sched     | No      | No   | Se usa para para guardar y restaurar intena                                    |
| yield     | No      | Si   | Cuando se acaba el quantum se la llama, decrementamos prio                     |
| sleep     | No      | Si   | Cuando ocurre una interrupcion pero no se acaba el quantum, incrementamos prio |
| wakeup    | No      | No   | Solo despierta a los procesos dormidos, nuestros registros no influyen         |
| kill      | No      | No   | Solo pone un proceso sleep en run para que haga release                        |
| procdump  | Si      | Si   | La utilizamos para imprimir los valores                                        |
